### Q6 — What is Run Chat?

Before implementation, the user asked:

> Before implementing anything tell me in details what is run chat, what it does and how it works

Answer given:

- Run Chat is the Research Console's per-run Q&A surface.
- It lets a user ask questions about a specific completed research run.
- It is not part of the research artifact.
- It does not rewrite `report.json`, create Sources, affect Evidence Quality, or create/score
  Predictions.
- It loads selected run artifacts into bounded prompt context, sends the user question plus recent
  chat turns to the configured model provider, and returns a grounded answer.
- It stores transcript only in browser `localStorage`, not under `data/runs/`.
- Current frontend: `app/client/components/run-chat.svelte`.
- Current backend: `app/chat.ts`.
- Provider setup: `app/server.ts`.
- Codex live search passes `-c tools.web_search=true` and `-c web_search=live` when active.
- Boundary: web findings are ephemeral context, not persisted Sources.
