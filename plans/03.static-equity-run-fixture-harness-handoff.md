## Approved plan (verbatim)

### Context

We want to iterate on the equity pipeline (collector, normalization, source-plan, orchestration,
report assembly) **without paying for or waiting on live APIs**, while still being able to judge
whether prompt/skill/playbook changes actually improve the research.

The user's overriding requirement: **test all internal functionality for real; only mock the two
true external boundaries** — HTTP `fetch` and the LLM `ModelProvider`.

### Decisions locked in during grilling

1. **Boundary = fetch + LLM only.** Replay recorded HTTP responses through `fetchImpl`; run the
   **real** collector, cache, normalization, source-plan, orchestration, report assembly.
2. **LLM is a toggle over one shared data-replay foundation:**
   - **Regression mode (default, CI, $0, deterministic):** replay recorded LLM outputs.
   - **Eval mode (on-demand, local, costs $):** live `ModelProvider`, so real prompt/skill/playbook
     changes are exercised. Data still replayed (free/fast/deterministic inputs).
3. **LLM cassette keyed by `stage (+ model id) + call index` (lenient),** not prompt-content hash —
   so pipeline changes that shift evidence don't invalidate the cassette and force a paid re-record.
4. **Assertions: golden snapshot + invariant asserts.** Snapshot the full deterministic run output
   (report.json, normalized/*, analytics, report.md) with volatile fields scrubbed; the diff is the
   "did results change" signal. Plus a few explicit invariant checks.
5. **Recording: a dev-only script that wraps both seams in one live run,** scrubs secrets, writes
   keyed cassettes + metadata. One command to (re)generate a fixture.
6. **First slice: brief + deep AAPL** (deep pulls in peer-universe LLM inside `collectSources`,
   evidence/web-gather loops, and forecast-disagreement challenger models).

### Non-goals

- Not replacing live smoke/provider-health checks (`src/health/validation.ts`).
- Not asserting synthesis *quality* in CI — that's eval mode + `run-review`.
- No `CollectedSources`-injection tier in this slice.

### Architecture

```
record-fixture-run (live, once)          replay (regression = CI; eval = local)
  real fetch  ──capture──▶ data cassette   ──▶ fetchImpl replay ─▶ REAL collector/cache/normalize
  real LLM    ──capture──▶ llm cassette    ──▶ provider: replay (regression) | live (eval)
                                                        │
                                          REAL orchestrator ─▶ real report assembly / validation
                                                        │
                                    golden snapshot (scrubbed) + invariant asserts
```

Two external seams, both already injectable:
- **Data:** `collectSources(..., fetchImpl)` (`src/sources/collector.ts:615`) AND the orchestrator's
  post-collection `sourceFetchImpl` (`RunResearchJobInput`, `src/research/orchestrator.ts:84`). Both
  must be fed the *same* data cassette.
- **LLM:** `RunResearchJobInput.provider` (single `ModelProvider`, `src/model/types.ts`), plus the
  `PeerUniverseSeam.provider` threaded into `collectSources` for deep runs (`collector.ts:588`).

### Components to build

**1. Data cassette + replay fetch — `tests/support/run-fixtures/data-cassette.ts`**
- Cassette = map keyed by `canonical(url) + method + bodyHash` → `{ status, headers(minimal), body }`.
  Reuse `canonicalRequestUrl` / `makeCacheKeyForTest` (`src/sources/cache.ts:74,138`). Export
  `canonicalRequestUrl` (or a small helper) rather than duplicating.
- `makeReplayFetch(cassette): FetchLike` — hit → real `Response`; miss → loud error naming the
  request (mirror `tests/support/setup.ts:14`).
- Replay sits **below** the cache so the real cache write/read path runs. Pin `now` from metadata so
  `cacheStatus` is `current`.

**2. LLM cassette + replay provider — `tests/support/run-fixtures/llm-cassette.ts`**
- Ordered list per key `stage (+ model id)`; replay pops by incrementing call index. A recorded
  evolution of `providerReturning` (`tests/support/mocks.ts:24`).
- **Loop overflow fallback:** on index overflow (evidence-request/web-gather drift) return a
  recorded terminal/empty output and log — never throw.

**3. Record script — `scripts/record-fixture-run.ts` (dev-only, not CI)**
- `bun run scripts/record-fixture-run.ts equity-aapl-brief AAPL --brief` and `... equity-aapl-deep AAPL --deep`.
- Runs real CLI path (`src/app.ts` DI), capturing every fetch + `provider.generate`.
- **Secret scrubbing:** URLs through `canonicalRequestUrl`; drop/replace auth headers (SEC
  `User-Agent`, Tradier `Authorization`, Exa/Massive `x-api-key`); assert no known token substrings
  remain before writing.
- Metadata: pinned `now`/`generatedAt`, exact command, `challengerModels`, code version. Reuse
  `compactOversizedRawSnapshots` for oversized bodies. Needs live keys + costs LLM $ once per record.

**4. Harness / loader — `tests/support/run-fixtures/index.ts`**
- `loadFixture(name)` → `{ dataCassette, llmCassette, meta }`.
- `runFixture(name, { llm: "replay" | "live" })` → isolated `AppConfig`, both fetch seams wired to
  the data cassette + peer-universe provider, provider per mode, runs `persistResearchJob`, returns
  run dir + parsed artifacts.
- State isolation: temp `dataDir`, temp `newsSeenPath`, temp `peerUniverseLearnedPath`,
  `historyOptions.recentDays` pinned, pinned `now`.

**5. Regression test — `tests/equity-fixture-run.test.ts`**
- For brief + deep: `runFixture(name, { llm: "replay" })`, then:
  - Golden snapshot with volatile fields scrubbed (generatedAt, completedAt, durationMs, token/cost,
    temp paths) — includes `report.md`.
  - Invariant asserts (reuse `tests/golden.test.ts`): predictions observable (ADR 0004); research-
    only safety passes (ADR 0001); exactly one disclaimer; source-plan/evidence-lanes/source-gaps/
    analytics present + schema-valid; deep-only: forecast-disagreement present with expected
    challenger count.
- Document snapshot-update flow (`bun test --update-snapshots`) as the intentional-change path.

**6. Eval mode — `scripts/replay-fixture-run.ts` (dev-only)**
- `bun run scripts/replay-fixture-run.ts equity-aapl-deep --live` → data replayed, LLM live, writes a
  labeled run dir under `data/runs/` (or temp) for the `run-review` skill to diff.

### Risks / edge cases

- **Deep loop drift** → lenient index + terminal-fallback (component 2).
- **Cassette staleness** → loud miss errors + one-command re-record (costs LLM $ once, needs keys).
- **Golden churn** → intentional changes require snapshot update; volatile-field scrubbing keeps
  diffs meaningful.
- **Secret leakage** → record script asserts no known token substrings survive scrubbing.
- **Deep config pinning** → `challengerModels`, evidence/web-gather budgets, and `now` must be
  identical between record and replay or artifacts diverge.

4. `bun run scripts/replay-fixture-run.ts equity-aapl-deep --live` → run dir; diff two eval runs via
   `run-review`.

## Full grilling Q&A (decision record)

**Q1 — Where should the fixture boundary sit for the primary harness?**
Options: (a) Fetch + LLM only (full pipeline); (b) inject `CollectedSources` (doc 03 Tier B);
(c) both, fetch-seam primary.
**User answer:** "Only the external data fetches should be mocked. I wish that the LLM responses are
live tested this way I also test changes to any prompts, skills, etc... Or do you think this is not
recommended?"

**Q2 — Given the cost/determinism trade-off, how should the LLM boundary be handled?**
Reasoning surfaced: the LLM is the *dominant* recurring cost (data providers are mostly free), live
LLM is non-deterministic (breaks "did results improve" diffs), needs secrets in CI, and only half-
helps speed. Options: (a) toggle — replay for CI, live for eval; (b) live always; (c) replay always.
**User answer:** (a) **Toggle: replay for CI, live for eval.** Live LLM retained as an on-demand eval
mode, not the CI default.

**Q3 — How should the replay-LLM cassette be keyed (regression mode)?**
Options: (a) stage + call index (lenient); (b) prompt-content hash (strict — any upstream pipeline
change forces a paid re-record); (c) hybrid hash-with-stage-fallback.
**User answer:** (a) **Stage + call index (lenient).**

**Q4 — What should the regression test assert against the deterministic run output?**
Options: (a) golden snapshot + invariant asserts; (b) targeted asserts only (doc 03 Tier A);
(c) snapshot deterministic artifacts only (exclude report.md).
**User answer:** (a) **Golden snapshot + invariant asserts.**

**Q5 — How should fixtures/cassettes be recorded and refreshed?**
Options: (a) record script wrapping both seams; (b) hand-author; (c) derive from an existing run dir
(lossy on request keying).
**User answer:** (a) **Record script wrapping both seams.**

**Q6 — What's the scope of the first implementation slice?**
Options: (a) brief equity AAPL only; (b) brief + deep AAPL; (c) brief AAPL + international brief.
**User answer:** (b) **Brief + deep AAPL** (accepts the larger surface: peer-universe LLM in
`collectSources`, evidence/web-gather loops, forecast-disagreement).
